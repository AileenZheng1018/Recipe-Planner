# Recipe Planner — 一步一步来

按顺序做下面的步骤就行。

---

## 第一步：先试一下「规划」功能（不用登录）

1. 打开「终端」（Mac 在「启动台」里找「终端」）。
2. 输入下面这一行，按回车（整行复制粘贴即可）：

   ```bash
   cd "/Users/apple/Desktop/Recipe Planner"
   ```

3. 再输入下面这一行，按回车：

   ```bash
   python3 agent.py
   ```

4. 终端里会出来「一周餐单」和「购物清单」。
   - 如果出来了 → 说明规划功能正常，可以继续下一步。
   - 如果报错说找不到 python3 → 需要先安装 Python（可以告诉我你的电脑系统，我帮你写安装步骤）。

---

## 第二步：注册一个 Supabase 账号（用来存你的用户和库存）

1. 用浏览器打开：https://supabase.com
2. 点右上角 **Sign in**，用 Google 或邮箱注册/登录。
3. 登录后点 **New project**（新建项目）：
   - **Name** 随便填，比如：`recipe-planner`
   - **Database Password** 自己设一个密码，记下来（以后会用到）。
   - 下面的区域可以默认，直接点 **Create new project**。
4. 等一两分钟，项目建好后会进到项目页面。

---

## 第三步：在 Supabase 里建好「用户表」和「库存表」

1. 在 Supabase 项目页面左侧，点 **SQL Editor**。
2. 点 **New query**（新建查询）。
3. 打开你电脑上的这个文件（用记事本或 VS Code 都行）：
   - 路径：`Recipe Planner` 文件夹里的 `supabase` 文件夹，里面的 `schema.sql`。
4. 把 `schema.sql` 里的**全部内容**复制下来。
5. 回到 Supabase 的 SQL 输入框，粘贴进去。
6. 点右下角 **Run**（或按 Ctrl+Enter / Cmd+Enter）。
7. 如果下面显示绿色成功、没有红色报错 → 这一步就完成了。

---

## 第四步：拿到 Supabase 的「地址」和「钥匙」

1. 在 Supabase 左侧点 **Settings**（齿轮图标）。
2. 再点 **API**。
3. 你会看到两行重要的东西，先别关这个页面：
   - **Project URL**：一长串，像 `https://xxxxx.supabase.co`
   - **Project API keys** 下面有一个 **anon** **public**：一长串字母数字，点一下可以复制。

---

## 第五步：在前端项目里填上「地址」和「钥匙」

1. 打开「终端」，输入（可以整行复制）：

   ```bash
   cd "/Users/apple/Desktop/Recipe Planner/web"
   ```

2. 再输入（复制整行）：

   ```bash
   cp .env.example .env
   ```

3. 用记事本或 VS Code 打开 `Recipe Planner/web` 文件夹里的 **`.env`** 文件。
4. 把里面的两行改成你自己的（不要删掉 `VITE_` 前缀）：
   - 第一行：把 `https://your-project.supabase.co` 换成你在第四步看到的 **Project URL**。
   - 第二行：把 `your-anon-key` 换成你在第四步复制的 **anon public** 那一串。
5. 保存 `.env` 文件。

---

## 第六步：安装依赖并打开网页

1. 终端里确保已经在 `web` 目录（如果关了终端，重新执行一次第五步的第 1 行）。
2. 输入并回车：

   ```bash
   npm install
   ```

3. 再输入并回车：

   ```bash
   npm run dev
   ```

4. 终端里会出现一行类似：
   - `Local: http://localhost:5173/`
5. 用浏览器打开这个地址（复制到浏览器地址栏回车即可）。

---

## 第七步：在网页上注册和登录

1. 打开 `http://localhost:5173/` 后，如果跳转到登录页，点「没有账号？注册」。
2. 输入你的**邮箱**和**密码**，点「注册」。
3. 去你的邮箱里找 Supabase 发的确认邮件（有的项目会要求确认），点链接确认（如果没收到，先看垃圾邮件）。
4. 回到 `http://localhost:5173/`，用同一邮箱和密码点「登录」。
5. 登录成功后会看到「我的库存」页面（一开始是空的就对了）。

---

## 到这里你已经完成的内容

- 规划功能：用 `python3 agent.py` 可以出餐单。
- 网页：可以注册、登录，并且你的信息存在 Supabase 里。
- 库存表已经建好，之后可以在网页上做「添加/编辑库存」功能（或继续用 Agent 默认库存）。

---

## 常见问题

- **终端里说 command not found: python3**  
  需要先安装 Python，装好后重新打开终端再试第一步。

- **终端里说 command not found: npm**  
  需要先安装 Node.js（从 https://nodejs.org 下载安装），装好后重新打开终端再试第五、六步。

- **网页一直打不开或报错**  
  检查第五步的 `.env` 里 **Project URL** 和 **anon key** 是否填对、没有多空格、没有删掉 `VITE_` 前缀。

- **SQL 执行报错**  
  确认是复制了**整个** `schema.sql` 文件内容，并且是在 Supabase 的 **SQL Editor** 里点的 Run。
